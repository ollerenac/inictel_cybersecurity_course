# ISMS — Gestión de Seguridad de la Información

| Campo | Valor |

## ¿De qué trata esta categoría?

ISMS (Information Security Management System) agrupa ejercicios relacionados con la **gestión, cumplimiento y auditoría de seguridad** — no la explotación técnica, sino la comprensión de marcos normativos, análisis de riesgos, y evaluación de controles de seguridad.

Estos ejercicios son relevantes para roles como analista de seguridad, oficial de seguridad (CISO adjunto), o auditor de cumplimiento.

## Prerrequisitos recomendados

- **Conceptos de seguridad**: CIA triad (Confidencialidad, Integridad, Disponibilidad)
- **Redes básicas**: topología, puertos, protocolos comunes (HTTP, FTP, SSH, SMTP)
- No se requiere experiencia en explotación técnica — estos ejercicios son más analíticos que prácticos

## Conceptos clave

| Concepto | Qué es |
|----------|--------|
| **ISO 27001** | Estándar internacional para sistemas de gestión de seguridad de la información |
| **Análisis de riesgos** | Identificar, evaluar y priorizar riesgos de seguridad |
| **Control de seguridad** | Medida técnica u organizativa que reduce un riesgo |
| **Auditoría** | Evaluación sistemática del cumplimiento de políticas y estándares |
| **Gestión de vulnerabilidades** | Proceso de identificar, clasificar y remediar vulnerabilidades |
| **CVSS** | Common Vulnerability Scoring System — escala de severidad de vulnerabilidades (0–10) |

## Herramientas de esta categoría

| Herramienta | Uso principal |
|-------------|---------------|
| **nmap** | Descubrimiento de red y enumeración de servicios |
| **Nessus / OpenVAS** | Escáneres de vulnerabilidades para auditorías de red |
| **Metasploit (módulos de scan)** | Verificación de vulnerabilidades conocidas |
| **Documentación ISO 27001/27002** | Referencia de controles y políticas |
